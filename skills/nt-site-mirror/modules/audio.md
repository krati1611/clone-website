# Audio Module

Triggers: background music, ambient audio, soundscapes, hover/click/UI sounds, menu sounds, scroll-linked audio, audio-reactive visuals, provider-hosted audio tracks, Web Audio API, spatial audio, mute/volume controls.

Treat audio as part of the requested baseline. Never silently remove or downgrade an audible source feature.

## Discovery additions

Document the observed sources and providers; playback and autoplay behavior; mute/unmute and volume controls; triggers; synchronization with animations or interactions; responsive behavior; and browser autoplay restrictions. Mark unexercised states as unknown.

## Baseline construction

### Static Mirror

Preserve authorized audio dependencies and their deployed behavior where technically possible. Classify each observed audio system in the Asset Preservation Table. Keep provider-hosted or protected media external when it cannot be localized lawfully or technically.

### Editable Recreation fallback

When Static Mirror is unsuitable, recreate only the audio behavior needed for the requested faithful copy, based on recorded source evidence. Record any approximation and its fidelity impact before using it.

## Validation additions

For the declared routes, viewports, and interaction states, compare playback, timing, triggers, synchronization, mute/unmute, volume controls, responsive behavior, and autoplay-restriction handling with the source. Report blocked or unexercised states rather than inferring success.
